## 5. ソフトウェアユニット受入基準(箇条 5.5.3)

各ユニットは以下の基準を **全て** 満たすことを受入条件とする:
1. 詳細設計書の公開 API と実装が一致している
2. コーディング規約違反がない(指摘ゼロまたは是正済み)
3. 静的解析の警告がない(または正当化済み)
4. ユニット試験が全件合格している

## 6. 追加のユニット受入基準(箇条 5.5.4 ― クラス C)

クラス C では以下を追加の受入基準とする:

- [ ] **正常系の動作確認:** 正常入力で期待出力が得られること
- [ ] **境界値試験:** 入力値域の最小・最大・境界±1
- [ ] **異常系・エラー入力:** 値域外、NULL、不正状態からの呼出
- [ ] **資源使用:** スタック使用量、ヒープ確保量、実行時間が設計制約内
- [ ] **制御フロー網羅:** 分岐網羅(decision coverage){{%}} 以上
- [ ] **データフロー:** 未初期化変数・使用されない定義の排除
- [ ] **ハードウェア障害・ソフトウェア障害:** 検出と処置が設計どおり
- [ ] **並行処理:** 競合・デッドロック・優先度逆転への対処
- [ ] **タイミング:** タイムアウト・割込応答が規定時間内
