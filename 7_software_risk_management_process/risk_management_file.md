## 5. リスク推定(ISO 14971 箇条 5.5)

### 5.1 重大度と発生可能性の評価基準

#### 重大度(Severity)
| 段階 | 区分 | 例 |
|------|------|-----|
| S5 | 破局的 | 死亡、永続的重傷 |
| S4 | 重大 | 重傷、入院 |
| S3 | 重度 | 医療介入を要する傷害 |
| S2 | 軽度 | 一時的で軽微な傷害 |
| S1 | 無視可能 | 一時的な不快感 |

#### 発生可能性(Probability)
| 段階 | 区分 | 頻度の目安 |
|------|------|----------|
| P5 | 頻繁 | > 10⁻³/使用 |
| P4 | 可能性大 | 10⁻³〜10⁻⁴ |
| P3 | 時々 | 10⁻⁴〜10⁻⁵ |
| P2 | 稀 | 10⁻⁵〜10⁻⁶ |
| P1 | ほとんどない | < 10⁻⁶ |

### 5.2 リスクマトリクス(例)

|      | S1 | S2 | S3 | S4 | S5 |
|------|----|----|----|----|----|
| P5 | 中 | 高 | 高 | 許容外 | 許容外 |
| P4 | 低 | 中 | 高 | 高 | 許容外 |
| P3 | 低 | 中 | 中 | 高 | 高 |
| P2 | 無視 | 低 | 中 | 中 | 高 |
| P1 | 無視 | 低 | 低 | 中 | 中 |

許容方針:
- 「許容外」: 必ずリスクコントロール実施
- 「高」: ALARP 原則適用
- 「中」「低」: 必要に応じて低減

## 8. 全体的な残留リスクの評価(ISO 14971 箇条 8)

### 8.1 残留リスクの総合評価

- [ ] すべての特定ハザードに対して RCM が実装・検証された
- [ ] 個別残留リスクはすべて許容基準内である
- [ ] RCM 実装による新たなハザードは識別・評価された
- [ ] 医療機器の臨床的ベネフィットが残留リスクを上回ることを確認

## 10. 製造後の情報収集及びリスク再評価(ISO 14971 箇条 10)

- 情報源: 顧客フィードバック、不具合情報、同種機器の公開情報、規制当局情報、SOUP 公開異常情報
- 収集頻度: 月次(サマリ)、重大事象は発覚次第
- 再評価のトリガ:
  - 新たなハザードの識別
  - 発生頻度・重大度の想定超え
  - RCM の無効化・劣化
  - SOUP の重大脆弱性・不具合の公表
- 処置: 本 RMF 更新 → 関連ドキュメント(SRS、設計、試験)更新 → 箇条 7.4 に従いソフトウェア変更のリスクマネジメント実施
