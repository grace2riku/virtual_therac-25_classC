## 3. ソフトウェア要求事項を検証する試験の設定(箇条 5.7.1)

システム試験は、SRS のすべての要求事項が実装されていることを **試験により検証** する。検証方法が試験以外(レビュー・解析・デモ)である要求については、その根拠を記載する。

## 5. ソフトウェア問題解決プロセスの使用(箇条 5.7.2)

- 試験中に発見された不具合は、IEC 62304 箇条 9(ソフトウェア問題解決プロセス)に従って記録・分析・是正する。
- 是正に伴うソフトウェア変更は、影響範囲を評価し、該当試験を再実行する。
- 変更管理記録(変更 ID、影響評価、再試験結果)を残す。

## 6. 変更後の再試験(箇条 5.7.3)

- 試験中に発生したソフトウェア変更については、影響範囲を解析してシステム試験を選択的に再実行する。
- 安全関連機能・主要機能および変更に隣接する機能は、変更種別によらず必ず再実行する(回帰試験)。

## 7. ソフトウェアシステム試験の妥当性確認(箇条 5.7.4 ― クラス C)

試験自体の妥当性を以下の観点で確認する:

- [ ] 試験ケースが SRS を網羅している
- [ ] 試験手順・入力・期待結果が明確かつ再現可能である
- [ ] 試験環境が実使用環境を妥当に代表している
- [ ] 測定機器は校正されており、精度が要求を満たす
- [ ] 合否判定基準が客観的である
- [ ] 不具合検出能力が適切である
