>
> 本書は IEC 62304 に基づく医療機器ソフトウェア開発プロセスの学習・参考実装を目的とした **仮想プロジェクト** の成果物である。題材は 1985-1987 年に放射線過剰照射事故を起こした実在の医療機器 **Therac-25**(AECL)である。本書の記述は、当時の AECL 社内文書ではなく、公開された事故報告書・学術文献(Leveson & Turner 1993 ほか)を踏まえた **学習用の再構成** である。

## 1. 目的と適用範囲

### 1.1 目的
本書は、仮想 Therac-25 に搭載されるソフトウェア(TH25-CTRL)の開発プロセスを IEC 62304 の要求事項に従って計画し、その実施方法を定義することを目的とする。Therac-25 の事故要因(race condition、整数オーバフロー、ハードウェアインターロック廃止、独立レビュー不在 等)に対し、IEC 62304 のプロセスと成果物がどのように予防機能を発揮し得たかを、本書を起点に具体化する。

### 1.2 適用範囲
- 対象ソフトウェア: Therac-25 Virtual Control Software(TH25-CTRL)
- 対象バージョン: 0.1.0 以降
- 安全クラス: **C**(根拠は [SSC-TH25-001](../7_software_risk_management_process/software_safety_class_determination_record.md) を参照)
- 本計画は、ソフトウェア開発の開始からリリースまでのすべての活動に適用する。

## 3. ソフトウェア開発プロセスの概要(箇条 5.1.1)

### 3.1 ライフサイクルモデル
採用するライフサイクルモデル: **V字モデル(インクリメンタル方式)**

各インクリメントは Therac-25 の事故主要因に対応付けて構成し、単体試験→結合試験→システム試験→リスクマネジメントレビューまでの V字サイクルを一周する(AAMI TIR45 に準拠)。想定インクリメント:

| Inc. | 主要機能 | 対応する Therac-25 事故要因 |
|------|---------|------------------------|
| Inc.1 | ビーム生成・モード制御コア(電子線/X 線モード切替、エネルギー設定、磁場指令、ターンテーブル位置決め) | 電子ビーム高出力モードと X 線ターゲット非挿入の組合せ発生(HZ-001) |
| Inc.2 | インターロック/安全監視(ドア、ターンテーブル位置センサ、ドーズモニタ、ビーム電流) | インターロック検知失敗(HZ-004)、ドーズモニタ誤処理(HZ-008) |
| Inc.3 | タスク同期・タイミング保証(操作者入力確定→設定反映→ビームオンまでの排他制御) | race condition(HZ-002)、整数オーバフロー(HZ-003) |
| Inc.4 | 操作者 UI・エラー報告・ロギング(明示的エラー表示、バイパス禁止、治療履歴) | 暗号的エラー表示と操作者バイパス(HZ-006) |

## 6. ソフトウェア開発標準・方法・ツール計画(箇条 5.1.4 ― クラス B, C)

### 6.1 コーディング規約
- 使用言語: **C++20**(Inc.1 着手時に C++23 への移行可否を再評価)
- 適用規約: **MISRA C++ 2023** を第一候補、補助として **CERT C++ Secure Coding Standard**
- 静的解析ツール: clang-tidy(LLVM 17 以降推奨)、cppcheck、Include-What-You-Use
- 動的解析ツール: AddressSanitizer / UndefinedBehaviorSanitizer / **ThreadSanitizer**(Therac-25 race condition の再現・検出用、必須)

## 14. 共通ソフトウェア欠陥の識別と回避(箇条 5.1.12 ― クラス B, C)

選択した開発言語・技術・ツールに固有の、共通的に知られるソフトウェア欠陥類型を識別し、回避策を定義する。

| 欠陥類型 | 該当技術 | 回避策・検出手段 | Therac-25 との関連 |
|---------|---------|---------------|-------------------|
| バッファオーバフロー | C++ | MISRA C++ 準拠、`std::span` / `std::array` 活用、静的解析、ASan | 直接対応なし(関連低) |
| ヌルポインタ参照 | C++ | 静的解析、防御的チェック、`std::optional` / 参照渡し優先 | 関連低 |
| **整数オーバフロー(とくに周期カウンタ)** | 全言語 | 静的解析、安全な算術ライブラリ、境界値試験、`std::numeric_limits` 検証 | **Yakima 事故 Class3 変数と同型の欠陥**(HZ-003) |
| 資源リーク | C++ | RAII、コードレビュー、動的解析(ASan) | 関連低 |
| **並行処理不具合(競合・デッドロック)** | マルチスレッド(`std::thread`/`std::atomic`/`std::mutex`) | 設計レビュー、**ThreadSanitizer 必須**、モデル検査(TLA+)、排他領域の明示、lock-free アルゴリズムの慎重適用 | **Tyler 事故と同型の欠陥**(HZ-002) |
| 未初期化変数 | C++ | コンパイラ警告最高レベル(`-Wall -Wextra -Wpedantic -Werror`)、静的解析、`{}` 初期化の徹底 | 関連中 |
| 例外処理漏れ | C++ | 静的解析、ピアレビュー、`noexcept` の適切な付与 | 関連中 |
| **共有状態の競合的アクセス**(UI 割込みと制御ループの共有変数) | マルチタスク | 設計上の **分離(箇条 5.3.5)**、メッセージパッシング、イミュータブルデータ共有 | **Therac-25 アーキテクチャ上の根本欠陥**(HZ-002, HZ-003) |
| **旧コード再利用時の前提喪失** | あらゆるレガシーコード | 再利用前のハザード再解析、SOUP 扱い、ユニット試験の再実施 | **Therac-6/20 コード再利用の指摘事項**(HZ-007) |
| **エラー処理の曖昧化・バイパス許容 UI** | 全言語・UI 層 | エラー分類の明示、「継続」操作の制限、IEC 62366-1 準拠、妥当性確認試験 | **"MALFUNCTION 54" 問題**(HZ-006) |
| 浮動小数点誤差・丸め誤差(線量計算) | C++(`float`/`double`) | 精度要件に基づく型選択、単位変換の明示、境界値試験 | 関連中(HZ-005) |

参照: CWE、CERT Coding Standard、FDA Guidance(例: "Content of Premarket Submissions for Software Contained in Medical Devices")等。
