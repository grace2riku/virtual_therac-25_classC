- 構成管理の詳細は別書 **ソフトウェア構成管理計画書(SCMP)** を参照する。
- 構成管理ツール: Git(リポジトリ: {{URL}})

## 12. 管理対象となる支援項目(箇条 5.1.10)

構成管理下に置く支援項目の分類を以下に示す。具体的な項目・バージョンは **構成アイテム一覧(CIL)** に記録する。

## 13. 検証前の構成アイテムの制御(箇条 5.1.11)

- コードおよびドキュメントは、検証(レビュー・試験)に入る **前** に構成管理下に置く。
- 具体的実施方法: Pull Request の作成時点でブランチを保護リポジトリへ push し、レビュー承認前に他者が書き換えられないようにする。
- 検証済みベースラインへは、承認されたマージのみ反映する。

## 14. 共通ソフトウェア欠陥の識別と回避(箇条 5.1.12 ― クラス B, C)

選択した開発言語・技術・ツールに固有の、共通的に知られるソフトウェア欠陥類型を識別し、回避策を定義する。

| 欠陥類型 | 該当技術 | 回避策・検出手段 |
|---------|---------|---------------|
| バッファオーバフロー | C/C++ | MISRA-C 準拠、静的解析、境界チェック関数の使用 |
| ヌルポインタ参照 | C/C++ | 静的解析、防御的チェック |
| 整数オーバフロー | 全言語 | 静的解析、安全な算術ライブラリ |
| 資源リーク | 全言語 | RAII、コードレビュー、動的解析 |
| 並行処理不具合(競合・デッドロック) | マルチタスク | 設計レビュー、動的解析、形式手法 |
| 未初期化変数 | C/C++ | コンパイラ警告最高レベル、静的解析 |
| 例外処理漏れ | C++/Java | 静的解析、ピアレビュー |
