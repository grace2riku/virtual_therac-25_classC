## 5. ソフトウェア結合試験(箇条 5.6.3)

### 5.1 試験方針
- ユニット間インタフェース(IF-U-xxx)、外部インタフェース(IF-E-xxx)の動作を検証する。
- SRS との関連を保持し、試験ケースを SRS / RCM に紐付ける。

## 6. 結合試験の内容(箇条 5.6.4)

結合試験は以下の観点を含む:

- [ ] 要求された機能が結合された状態で実現されていること(機能試験)
- [ ] 外部機器・他ソフトウェアとのインタフェース(入力・出力)の動作
- [ ] 内部インタフェース(ユニット間のデータ授受)の動作
- [ ] リスクコントロール手段(RCM)が結合状態で機能すること
- [ ] タイミング、同時実行、並行処理、割込の取扱い
- [ ] 異常条件(ハードウェア障害、異常入力、例外)への応答
- [ ] 予想される運用条件下での挙動
- [ ] 資源(メモリ、CPU、ディスク、通信帯域)の使用量

## 7. 結合試験手順の評価(箇条 5.6.5)

試験計画・手順は、レビューにより以下を確認する:

- [ ] 計画が SRS・アーキテクチャ設計書と整合している
- [ ] 各試験ケースが期待結果・合格基準を明示している
- [ ] 試験が再現可能である(環境・手順・入力データが文書化されている)
- [ ] リスクコントロール手段を検証するケースが含まれている
