## 3. 構成識別(箇条 8.1)

### 3.1 構成アイテム識別手段の確立(箇条 8.1.1)

すべての構成アイテムは、以下の識別情報をもつ。

- 一意の識別子(CI ID)
- 名称
- 種別(ソースコード / ドキュメント / SOUP / ツール / 試験資産 等)
- バージョン(ベースラインまたはコミットハッシュ)

識別の実現手段:
- バージョン管理システム: Git(SHA-1 コミットハッシュ + タグ)
- リリースタグ規則: `v{{MAJOR.MINOR.PATCH}}`
- ドキュメント: ファイル冒頭に **ドキュメント ID + バージョン** を明記
- 構成アイテム一覧は本書と同ディレクトリの [`configuration_item_list.md`](./configuration_item_list.md) で一元管理する

## 4. 変更制御(箇条 8.2)

### 4.1 変更要求の承認(箇条 8.2.1)

すべての構成アイテムへの変更は、承認された変更要求(CR: Change Request)に基づいて実施する。

| CR 区分 | 承認者 |
|--------|-------|
| 軽微(誤字、コメント) | {{氏名}} |
| 中程度(非安全関連機能) | CCB(変更管理委員会) |
| 重大(安全関連、アーキテクチャ) | CCB + 品質責任者 + RA 責任者 |

変更要求の記録項目:
- CR ID、要求者、要求日
- 変更対象(CI ID)
- 変更理由(問題報告 ID、保守要求 ID 等)
- 影響範囲解析結果
- リスクマネジメント評価(箇条 7.4)
- 承認者、承認日

#### 4.1.1 事後記録の取り扱い(軽微な台帳更新)

既に承認・実装・検証・マージ済みの CR / PR に対する **事後記録のみ** の変更は、新規 CR の起票対象とせず、元 CR に付随する記録行為として扱う。対象例:

- 変更要求台帳(CRR)の状態遷移・クローズ日の記入
- 構成アイテム一覧(CIL)のバージョン昇格反映
- リリースノート・マイルストーン・ベースラインラベルの付与

これらの事後記録を通常の CR / PR フローで処理すると「CR クローズ用の CR」が必要となり循環が生じる。この循環を避けるため、次の条件を **すべて** 満たす場合に限り、`main` への直接 push(Branch Protection の admin bypass)を許容する。

1. 変更内容が、既に承認・検証済み CR の事後記録のみであること(仕様・設計・実装・検証手順の変更を含まない)
2. 事前にローカルで CI 相当の lint(`npx markdownlint-cli2 "**/*.md"`)を実行し 0 エラーを確認していること
3. コミットメッセージに元 CR ID と事後記録である旨を必ず記載すること(例: `chore(CR-0042): close CRR after merge`)
4. bypass 操作は GitHub audit log に記録される。**リリース前 CCB 監査** および **各ベースライン確定時** に bypass 回数・対象・操作者を必ずレビューし、濫用を防ぐこと

本ルートは軽微な事後記録のみを対象とし、要求事項・設計・実装・試験結果への変更には適用しない。適用時は CR 台帳の備考欄に「bypass 使用」と明記する。

### 4.2 変更の実装(箇条 8.2.2)

承認後の変更実装は以下のルールに従う。

- フィーチャブランチで作業、Pull Request を経由して統合
- コードレビュー必須(レビュアは作成者以外)
- コミットメッセージに CR ID を含める
- 静的解析・ユニット試験は CI で自動実行

### 4.3 変更の検証(箇条 8.2.3)

変更は、その規模と安全性影響に応じた検証を経て受け入れる。

| 変更規模 | 最低限の検証 |
|---------|-----------|
| 軽微 | コードレビュー、該当ユニット試験 |
| 中程度 | 上記 + 結合試験 + 該当 RCM 検証 |
| 重大 | 上記 + システム試験 + リスク再評価 |

### 4.4 変更のトレーサビリティ(箇条 8.2.4)

以下の連鎖を維持する。

```
問題報告/変更要求 → CR → コミット → マージ → ベースライン → リリース
```

具体的な手段:
- コミットメッセージに CR ID(例: `CR-0123: ...`)
- PR 本文に問題報告 ID・関連 SRS・関連 RCM を記載
- リリースノートに含まれる CR を列挙
