## 1. 目的と適用範囲

### 1.1 目的

本書は、仮想 Therac-25(TH25-SIM-001)に搭載される **Therac-25 Virtual Control Software**(TH25-CTRL)のソフトウェア要求事項を IEC 62304 箇条 5.2 に基づいて定義し、後続のアーキテクチャ設計(SAD)・詳細設計(SDD)・実装・試験(UT/IT/ST)の基礎とする。

本書は、Therac-25 で発生した 6 件の放射線過剰照射事故(Tyler / Hamilton / East Texas / Yakima 等、1985-1987)の主要因(電子モードと X 線ターゲット位置の不整合、race condition、整数オーバフロー、暗号的エラー UI、ハードウェアインターロック廃止、レガシーコード再利用前提喪失)に対する **ソフトウェア側のリスクコントロール手段(RCM)を要求事項として明示** する。

### 1.2 適用範囲

- 対象ソフトウェア: TH25-CTRL(C++20 で実装される仮想シミュレータ制御ソフトウェア)
- 対象バージョン: 0.2.0 以降(本 v0.1 SRS は Inc.1 範囲、SRS v0.2 で Inc.2 範囲を追加予定)
- 安全クラス: **C**(根拠は SSC-TH25-001 v0.1 §6 を参照)
- スコープ外: 実機ビーム制御・実物理ハードウェア(本プロジェクトは PC 単体で動作する仮想シミュレータのみを対象)

### 1.3 用語と略語

本書で使用する略語は CLAUDE.md「英語略語の統一」表に従う。Therac-25 固有用語:

| 用語 | 定義 |
|------|------|
| 治療モード(TreatmentMode) | Electron(電子線)/ XRay(X 線)/ Light(視認光、患者位置決め用)の 3 種 |
| ターンテーブル(Turntable) | 治療モードに応じてビーム経路上に配置される構造物(電子線=空、X 線=ターゲット+フラットニングフィルタ、視認光=フィールドミラー)を切り替える回転台 |
| ベンディング磁石(Bending Magnet) | 電子ビームを患者方向に偏向させる電磁石。エネルギー設定に応じて電流値を変える |
| イオンチェンバ(Ion Chamber) | ドーズモニタ。Therac-25 は 2 組を冗長配置 |
| 強い型(Strong Type) | 単位や意味を型として表現し、暗黙変換による単位混在を静的に防ぐ C++ 型(例: `Energy_MeV`、`DoseUnit_cGy`) |

## 3. システム要求事項からソフトウェア要求事項への展開(箇条 5.2.1)

本プロジェクトは仮想プロジェクトであり、対応する実機システム要求仕様書(SyRS)は別途存在しない。代わりに **Therac-25 の公知仕様**(参考文献 [15] 等)から派生したシステムレベルの擬制要求(`SYS-NNN`)を設定し、本書の SRS 要求事項を割り付ける。割付は §9 トレーサビリティマトリクスに示す。

擬制システム要求(主要のみ抜粋):

| ID | システム要求 |
|----|-----------|
| SYS-001 | 患者に処方された治療モード(電子線または X 線)・エネルギー・ドーズで治療を行えること |
| SYS-002 | 操作者の入力誤り・装置内部の単一故障により患者に過剰被曝を与えないこと |
| SYS-003 | 装置異常時は速やかにビームを停止し、操作者に明確な警告を提示すること |
| SYS-004 | 操作・治療履歴を改ざん不可能な形で記録すること |
| SYS-005 | 治療装置として求められる規制要求(IEC 60601-2-1 等)に準拠すること |

## 4. ソフトウェア要求事項(箇条 5.2.2)

### 4.1 機能及び能力要求事項(箇条 5.2.2 a)

#### 4.1.1 治療モード制御(Inc.1 中核)

| ID | 要求事項 | 種別 | 優先度 | 検証方法 | 関連 |
|----|---------|------|-------|---------|------|
| SRS-001 | TH25-CTRL は 治療モードを `enum class TreatmentMode { Electron, XRay, Light }` で型安全に管理する。整数値や文字列での代替表現を禁止する | 機能 | 必須 | レビュー + 試験(型システムによる強制) | RCM-001 / HZ-001 |
| SRS-002 | TH25-CTRL は 治療モード変更指令を受信した際、以下の遷移シーケンスを必ず実行する: (1) 現在ビームオフであることの確認(成立しなければ拒否)(2) ターンテーブル位置指令の発行 (3) 位置センサ 3 系統の到達確認(全系統が目標位置 ±0.5 mm 以内)(4) ベンディング磁石電流の目標値到達確認(±5% 以内)(5) 100 ms 以上の安定待機 (6) ビームオン許可フラグの設定 | 機能 | 必須 | 試験(ステップごとの状態網羅) | RCM-001 / RCM-005 / HZ-001 |
| SRS-003 | TH25-CTRL は ビームオン要求受信時、現在の治療モードと、ターンテーブル位置センサ 3 系統の報告位置の整合を再確認する。不整合を検出した場合は `ModePositionMismatch` エラーで拒否し、ビームオンを行わない | 機能 | 必須 | 試験(整合性網羅 9 通り = 3 モード × 3 位置) | RCM-001 / RCM-005 / HZ-001 |
| SRS-004 | TH25-CTRL は エネルギー設定値を以下の範囲・ステップで管理する: 電子線 1〜25 MeV(1 MeV ステップ)、X 線 5〜25 MV(1 MV ステップ)。範囲外の指令は `EnergyOutOfRange` エラーで拒否する | 機能 | 必須 | 試験(境界値: 0/1/25/26 MeV、4/5/25/26 MV) | RCM-008 / HZ-005 |
| SRS-005 | TH25-CTRL は ベンディング磁石電流指令を強い型 `MagnetCurrent_A` で扱い、現在のエネルギー設定との対応表(SDD で詳細化)に基づき計算する。型変換は `static_cast` 等の暗黙変換を禁止し、明示的な変換関数を経由する | 機能 | 必須 | レビュー(型システム) + 試験(対応表検証) | RCM-008 |
| SRS-006 | TH25-CTRL は ターンテーブル位置を強い型 `Position_mm` で扱い、所定 3 位置(Electron=0.0 mm、XRay=50.0 mm、Light=-50.0 mm)からの偏差が ±0.5 mm 以内であることを到達条件とする | 機能 | 必須 | 試験(境界値: 偏差 0.49/0.50/0.51 mm) | RCM-001 / RCM-005 |

### 4.4 ソフトウェア駆動のアラーム・警告・操作者メッセージ(箇条 5.2.2 d)

IEC 60601-1-8 を参照しつつ、本仮想プロジェクトでは音響アラームは扱わず、UI への視覚通知 + ログ記録に統一する。

| ID | アラーム/メッセージ | トリガ条件 | 優先度 | 通知方法 | 関連 |
|----|------------------|----------|-------|---------|------|
| SRS-ALM-001 | モード/位置不整合検知 | ターンテーブル位置センサ 3 系統のいずれかが、現在の治療モードと整合しない位置を報告 | 高 | 即時ビームオフ + 操作者 UI に高優先度警告 + ログ記録 | SRS-003 / RCM-001 / HZ-001 |
| SRS-ALM-002 | ドーズ目標超過 | 積算ドーズが目標値の 1.05 倍超(5% 超過マージン) | 高 | 即時ビームオフ + 操作者 UI に高優先度警告 + ログ記録 | SRS-009 / SRS-010 / RCM-008 / HZ-005 |
| SRS-ALM-003 | ターンテーブル位置センサ 3 系統不一致 | 3 系統センサ値の最大-最小差が ±1.0 mm 超 | 高 | ビームオフ + 操作者 UI に高優先度警告 + ログ記録 | SRS-I-005 / RCM-005 |
| SRS-ALM-004 | ベンディング磁石電流偏差 | 指令値と計測値の偏差が ±5% 超(連続 100 ms 以上) | 中 | 操作者 UI に中優先度警告 + ログ記録(継続可。重畳条件で高昇格) | SRS-I-007 |
| SRS-ALM-005 | 起動時自己診断失敗 | SRS-012 の確認のいずれか不成立 | 高 | 治療開始拒否 + 操作者 UI に詳細エラーメッセージ表示(現象 + 原因 + 推奨対処、SRS-UX-001) | SRS-012 / RCM-013 |

| ID | リスクコントロール手段(RCM) | 対応するハザード | 関連 SRS | 検証方法 |
